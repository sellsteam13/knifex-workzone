Основная информация

Для определения результата каждой игры (режим Mines) используются случайные числа, созданные генераратором доказуемых и абсолютно честных случайных чисел.

Верификация исхода игры пользователями происходит полностью детерминированным способ. StepX100 не имеет возможности как либо влиять на исход игры и манипулировать результатами в свою пользу, т.к. алгоритм предполагает использование такого параметра, как клиентский сид.

Генерация случайных чисел для режима Mines происходит с участием начального числа клиента, начального числа сервера и одноразового номера, что позволяет на выходе получать необходимое количество случайных чисел.

Каждый режим использует данный список рандомных чисел, чтобы определить его исход, преобразовав их в необходимый результат:

  • Перемешивание списка возможных результатов (mines)

Начальные данные:

За каждым игроком прикреплены следующие данные:

  • активный клиентский сид
  • серверный сид
  • одноразовый номер (nonce)

Следует учесть, что одноразовый номер увеличивается каждый раз после завершения любого раунда в каждом из режимов, которые используют данный генератор случайных чисел (режим Mines).

Клиентский сид:

Рандомный набор символов, который задается пользователем. По умолчанию используется строка, сгенерированная на стороне сервера.

Серверный сид:

Начальное число сервера генерируется сервером. Строка представляет собой произвольно сгенерированную шестнадцатеричную (0-9, a-f) строку из 64 символов. Для хэширования StepX100 использует алгоритм sha256 (википедия).

У каждого пользователя есть возможность сменить текущий серверный сид. Активация нового сида происходит сразу после смены его пользователем. При данном действии также сбрасывается одноразовый номер (nonce) на 0.

Затем предыдущий сид, который теперь неактивный, разворачивается и может быть проверен с помощью sha256.

Одноразовый номер (nonce):

Одноразовый номер представляет собой число, увеличивающееся на единицу каждый раз после завершения раунда, что позволяет генерировать уникальный вход для игры. Nonce является единственным параметром, который изменяется каждый раз после игры.

Конвертация входных данных в байты:
Все три перечисленных параметра (клиентский сид, серверный сид и одноразовый номер) используются в качестве входных данных для генератора рандомных чисел.

Байты генерируются с использованием функции HMAC_SHA512 (K, m) (википедия). Эта функция создает 64 случайных байта (0-255) из заданного начального серверного сида (K) и сообщения (m).
Сообщение создается с использованием начального числа клиента(клиентского сида), одноразового номера (nonce) и инкрементного числа.

Это число, начинающееся с 0, увеличивается на единицу каждый раз, когда используются 64 байта, возвращаемые функцией HMAC_SHA512 (K, m). Затем сообщение создается путем объединения начального числа клиента, двоеточия, одноразового номера, другого двоеточия и добавочного числа.HMAC_SHA512(server_seed, client_seed:nonce:round):
